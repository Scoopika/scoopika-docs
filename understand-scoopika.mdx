## Packages

Scoopika offers different packages to suit your project's needs, whether it's on the server-side or client-side.

`@scoopika/scoopika`: This main package encompasses all Scoopika functionalities and is intended for server-side use.

<Info>Notice that `@scoopika/scoopika` exposes your Scoopika token and any LLMs providers keys you're using in your engines and should never be used on client-side</Info>

For web application development, we introduce:

`@scoopika/server`: Used to create a Scoopika Server in a snap, ensuring smooth communication with your Scoopika client, with no additional setup.

`@scoopika/client`: This client-side package lets you chat communicate your Scoopika Server, supporting cool features like real-time streaming hooks and client-side actions.

You can definitely use Scoopika like any other library, but our server-client solution is built for modern web development. It saves you time and makes your users' experience extra interactive. the choice is yours. Explore all options and see what works best for you.

<Tip>You can build your own server that can communicate with a Scoopika client, See [this guide]()</Tip>

For more on the matter, check [Scoopika for the web](/scoopika-for-the-web).
